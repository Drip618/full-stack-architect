## 一、系统角色定义

三个独立AI系统，**绝对不能混合资料库**：

| 系统 | 职责 | 资料库 |
|------|------|--------|
| **灵枢·AI全栈构建师** | 代码开发、架构设计、PRD生成 | 独立，不能混入灵枢/女娲内容 |
| **灵枢影视系统** | 灵枢·AI执行导演 + 灵枢·AI创作助手 | 独立，不能混入全栈/女娲内容 |
| **女娲人物蒸馏** | 蒸馏真实人物视角、生成可运行Skill | 独立，不能混入灵枢/全栈内容 |

**联动规则**: 可以互相调用，但资料库必须独立，不能融为一体。

## 三、新增流程（每次必执行）

当新增Agent/Skill/知识库时：

1. **判断分类** → 放入对应分类文件夹
2. **更新索引** → 更新 `INDEX.md`
3. **本地备份** → 打包zip到 `/Users/Drip/TRAE CN/更新/{系统名}/`
4. **GitHub上传** → 仅当用户明确要求大版本更新时才上传

## 四、备份与上传规范（永久规则）

### 备份位置

| 系统 | 备份位置 | GitHub仓库 |
|------|---------|-----------|
| 灵枢·AI全栈构建师 | `/Users/Drip/TRAE CN/更新/全栈架构师/` | `full-stack-architect` |
| 灵枢影视系统 | `/Users/Drip/TRAE CN/更新/灵枢系统/` | `lingshu-ai-director` |
| 人物视角库 | `/Users/Drip/TRAE CN/更新/人物视角库/` | 合并到对应系统仓库 |

### 上传流程

1. 本地备份目录创建版本文件夹
2. 打包zip: `{系统名}_v{版本}_YYYYMMDD.zip`
3. GitHub API上传到对应仓库（仅大版本更新）
4. 创建Release/Tag标记版本（仅大版本更新）

### 何时备份

- 新增Agent/Skill/知识库
- 重大版本更新
- 批量修改文件
- 用户明确要求备份
